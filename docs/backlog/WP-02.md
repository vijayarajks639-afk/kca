# WP-02 — Contract package (Pydantic)

**Epic:** E1 Foundation · **Sprint:** Sprint 0 · **Package:** `contracts/` · **Depends on:** WP-01
**Branch:** `wp-02-contract-package-pydantic`

## Scope

DIP contract schema (paper §8.2), ledger event types (three clocks: valid time, record time, inference time), retrieval request/response envelope with as_of + caller identity (role, purpose, jurisdiction), abstention reason codes (MISSING_DECISION_RECORD, VERSION_CONFLICT, UNAUTHORISED_SOURCE, REDERIVATION_MISMATCH, AMBIGUOUS_TERM).

## Acceptance criteria (PR checklist — all must pass)

- [ ] Schemas versioned with a schema_version field
- [ ] JSON-schema export via a make target
- [ ] Round-trip (serialize/deserialize) tests for every schema
- [ ] No business logic in the package

## Rules reminder

Follow CLAUDE.md strictly: LLM in L3/L4 only · no LLM-computed numbers · retrieval takes as_of + caller identity, permission filter pre-ranking, fail-closed · every model call in the ledger · cross-package via contracts/ only · no cloud SDKs in core · tests ship with the WP.

## Out of scope

Anything not listed above. Do not refactor other packages, add dependencies beyond pyproject, or change contracts/ schemas without flagging it in the PR description.
