# WP-05 — Bitemporal knowledge store + as-of API

**Epic:** E2 Knowledge & context layer · **Sprint:** Sprint 1 · **Package:** `platform/knowstore/` · **Depends on:** WP-02, WP-03
**Branch:** `wp-05-bitemporal-knowledge-store`

## Scope

Valid-time + record-time on every corpus item using Postgres range types; as-of query API; version-conflict detection for a given date.

## Acceptance criteria (PR checklist — all must pass)

- [ ] As-of query returns the March policy even after the May revision exists
- [ ] Overlapping versions for one date raise VERSION_CONFLICT
- [ ] Migrations reversible

## Rules reminder

Follow CLAUDE.md strictly: LLM in L3/L4 only · no LLM-computed numbers · retrieval takes as_of + caller identity, permission filter pre-ranking, fail-closed · every model call in the ledger · cross-package via contracts/ only · no cloud SDKs in core · tests ship with the WP.

## Out of scope

Anything not listed above. Do not refactor other packages, add dependencies beyond pyproject, or change contracts/ schemas without flagging it in the PR description.
